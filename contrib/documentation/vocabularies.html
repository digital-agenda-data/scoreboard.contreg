The Semantic Web establishes foreign key relationships automatically, but you have to tell the system what vocabulary a given code is part of. The convention is to make the code into a URL by prefixing it with a fixed string. Best practice is then to place a description of the code at the URL location to ensure the URL isn't used for something else.

The prefix must always end with a slash (/) or a hash (#). Then the code. In the latter case (#), the code can't start with a number. The RDF specification disallows it.
