A core functionality of the Content Registry is to merge national Reportnet deliveries. The result is placed in your home folder as a compiled dataset on the website and it gives you the ability to query only the files that are merged.

How to merge (compile) deliveries

Click on the "Merge deliveries" on the left side. Choose an obligation, and potentially also a coverage period. You will then get a list of deliveries. Mark the ones that potentially will contain the files you need. Click on "Save files into dataset". Then a second page will appear asking you to select the files to include in the compilation. You select some or all and then choose the name of the dataset to save to.

There are three options:

  • Save into a new dataset
  • Add to an existing dataset
  • If you check the "Overwrite if dataset already exists" then an existing dataset will be emptied before the values are copied into it.

This means that the choice of files doesn't have to correct the first time. You can add more files at a later time.

How to recompile

Go to the dataset's special tab, select the source to remove and click "Recompile". The recompile will automatially run your gap-filling operations, so you might want to ensure they are activated/deactivated according to your needs.

How to add a leftout source

There are two ways:

  • You can find it via "Merge deliveries"
  • You can find the source, then use the drop-down menu to add it to a compiled dataset.
Tip

To find out if a source is already included in a compiled dataset, click on the "Resource references" tab of the source and check if there is a "generatedFrom" link from the compiled dataset.

How to remove a source

Go to the dataset's special tab, select the source to remove and click "Remove". This will not delete the source from the database - only remove the values from the compiled dataset that were copied from that source.

When you have removed a source it is strongly recommended to recompile the dataset.

What if a delivered file isn't available as RDF?

In this case you will have to download the file from CDR, fix the problem and upload it to your home directory on Content Registry. The site has the same conversion tools as CDR, so if it is just an issue of using the correct template then this should fix the problem. You then add the source to the compiled dataset.
